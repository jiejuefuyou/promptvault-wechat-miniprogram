# 全自动部署流程 — 你最少需要做的事

> 之前 README 里 5 步要你打开微信开发者工具点上传。这个文档是**全自动版本**。
>
> 你只做 4 件事——前 2 件**仅一次性**，后 2 件**每次发新版必做（API 不开放）**。

---

## 一次性 setup（共 ~30 秒，做 1 次永久不用再做）

### Setup 1: 关闭 IP 白名单（10 秒）

1. 浏览器打开 https://mp.weixin.qq.com 登录
2. 左侧菜单：**开发管理 → 开发设置**
3. 滚到「**小程序代码上传**」模块
4. 「IP 白名单」点**关闭**（或者点编辑加上当前机器 IP，但关闭最简单）

### Setup 2: 下载私钥文件（20 秒）

1. 同页面（小程序代码上传模块）
2. 「**小程序代码上传密钥**」→ 点**生成**
3. 弹出**只此一次**的下载提示 → 下载 `.key` 文件
4. 把这个文件拖到电脑上的：
   ```
   C:\Users\sh199\Desktop\autoapp\.secrets\wechat-mp-private.key
   ```
   （目录已存在，把下载的 .key 文件改名放进去即可）
5. ⚠️ 这个文件**只能下载 1 次**——丢了得重新生成（旧 key 自动作废）

**完成后告诉我「key 已就位」**——我立即跑首次 upload。

## 每次新版本（你只点 2 下，~30 秒/次）

CC 自动跑 `node tools/deploy.js` 上传 → 你做：

### Step 1: 提交审核（10 秒，1 click）

1. 浏览器 https://mp.weixin.qq.com → **管理 → 版本管理**
2. 「**开发版本**」找到刚 upload 的版本
3. 点**「提交审核」**
4. 备注随便填：「修复 / 新功能」

### Step 2: 发布（5 秒，1 click，审核通过后）

1. 1-3 天后审核通过，邮件通知你
2. 同页面：**审核版本** 找到通过的版本
3. 点「**发布**」

之后用户能搜到 / 扫码用。

## CC 后续会做的事（**你不用管**）

- 每次小程序代码改了 → CC 自动 commit + push GitHub
- CC 自动跑 `node tools/deploy.js` upload 体验版
- CC 在 STATUS.md 写一条「已上传 v2026.X.X」通知你
- 你看到通知 → 公众平台点「提交审核」（10 秒）
- 审核通过 → 点「发布」（5 秒）

## 为什么不能完全自动化

微信对个人账号有 3 项必须人工的限制：

| 操作 | 原因 |
|---|---|
| 下载 .key 文件 | 安全：私钥只能下载 1 次 |
| 关闭 IP 白名单 | 安全：用户必须确认开放上传 |
| 提交审核 | 政策：每次提审需要确认内容合规 |
| 发布 | 政策：审核通过后用户必须 confirm 发布 |

API 端发布在企业账号 + 「服务市场」 partner 才能拿到，**个人账号无法获取**。

## 你的总耗时对比

| 方案 | 一次性 setup | 每次新版本 |
|---|---|---|
| **传统**：每次开 IDE 上传 | 0 | ~3 分钟（打开 + 选项目 + 上传 + 填备注 + 提审 + 发布） |
| **本方案** | 30 秒 | ~30 秒（只点 2 下） |
| **节省** | — | **每次省 2.5 分钟** |

按每月 4 次更新算 = **每月省 10 分钟**。一年 2 小时。

但更核心的：**心智负担**——每次发版本不用打开 IDE / 等编译 / 担心配置漂移。
